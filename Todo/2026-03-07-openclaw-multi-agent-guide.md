# 一套 OpenClaw 程序运行多个 Agent（总结）

## 核心结论
最实用的有 3 种方式：

1. **多会话并行（推荐）**
   - 同一个 OpenClaw 实例里开多个 session。
   - 每个 session 绑定不同任务/频道/cron。

2. **子代理并行**
   - 主会话通过 `sessions_spawn` 拉起多个子会话（`runtime: subagent/acp`）。
   - 各子会话并行执行，主会话汇总结果。

3. **定时任务并行**
   - 给不同角色配置不同 cron job（文档执行者、代码执行者、巡检者）。
   - 任务互不阻塞。

## 推荐角色架构
- `main`：总控与对外回复
- `doc-executor`：文档与评审闭环
- `code-executor`：代码实现与测试
- `ops-checker`：健康检查/告警巡检

## 落地关键点
1. **职责隔离**：每个 Agent 只做一类事。
2. **工作目录隔离**：避免互相覆盖文件。
3. **消息通道明确**：定义谁往哪个频道汇报。
4. **状态落盘**：统一写入 `docs/*` 与 `memory/*`，保证可追踪、可接力。

## 适用场景建议
- 需求多线程推进：优先“多会话并行”。
- 需要主从调度：优先“子代理并行”。
- 需要周期自动化：优先“定时任务并行”。
