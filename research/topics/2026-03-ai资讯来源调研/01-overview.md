# 01-overview

## 调研主题

- 主题名称：AI 资讯来源站点调研（首批 5 个来源）
- 主题类型：技术项目 / 内容采集

## 背景与问题定义

- 当前背景：需要持续稳定地采集 AI 相关新闻与官方动态，用于资讯聚合与后续摘要分发。
- 核心问题：哪些站点适合作为首批高优先级来源，且抓取稳定、信息质量高。
- 触发原因：历史采集中部分 RSS endpoint 失效（404），需要统一改造为“RSS + WEB 混合”可用方案。

## 调研目标

- 主要目标（必须达成）：确定可用且稳定的首批 AI 资讯来源网站清单。
- 次要目标（可选优化）：给出每个来源的推荐采集方式（RSS / WEB）与风险提示。

## 成功标准（可验证）

- 标准 1：形成明确的来源清单（站点名 + URL + 采集方式）。
- 标准 2：每个来源都有接入建议与风险说明。
- 标准 3：能直接用于 `ai-global-news` 的 source 配置与后续扩展。

## 范围

- 包含：Google DeepMind、Anthropic、Azure AI Blog、The Verge AI、NVIDIA Blog。
- 不包含：社媒账号抓取、付费数据库、非公开 API 来源。

## 关键约束

- 时间约束：本轮先完成首批 5 个来源的可用方案。
- 预算约束：优先使用免费公开网页与 RSS。
- 资源约束：以现有采集器能力（RSS + WEB）为边界。
- 合规/政策约束：仅采集公开可访问页面，不绕过访问限制。

## 干系人

- 决策人：琦琦
- 执行人：赛博牛马
- 影响对象：资讯系统维护者、内容使用者

## 初始假设

- 假设 A：官方站点（DeepMind/Anthropic/Azure/NVIDIA）在“权威性”维度优于媒体转载。
- 假设 B：RSS 并非总是可用，WEB 抓取需要作为常驻兜底方案。

## 待验证问题清单

- [ ] 各来源长期稳定性（30 天维度）
- [ ] 反爬策略变化对抓取成功率的影响
- [ ] 采集去重策略在多来源并行时的效果
