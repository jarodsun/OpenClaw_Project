# HTML Slide 固化流程（Research）

目标：所有 `research/reports/*.md` 都能快速产出同级 HTML 演示稿。

## 一、标准输出位置

- 报告：`research/reports/<报告名>.md`
- 幻灯片：`research/reports/<报告名>.slide.html`（与 md 同级）

## 二、标准页结构（默认 7 页）

1. 封面（标题/时间/一句话结论）
2. 执行摘要
3. 关键数据与事实
4. 分析框架
5. 方案对比
6. 风险与应对
7. 结论与行动

> 需要扩展时：可加“附录/数据口径/Q&A”页。

## 三、从 md 到 slide 的映射

- `1. 执行摘要` -> 第 2 页
- `4. 关键发现` -> 第 3 页
- `5. 专项分析` -> 第 4 页
- `6. 方案对比` -> 第 5 页
- `7. 风险与应对` -> 第 6 页
- `8/9. 结论与行动` -> 第 7 页

## 四、制作步骤（每次执行）

1. 复制模板：
   - `research/templates/slide/report-slide-template.html`
2. 命名为同级输出：
   - `<报告名>.slide.html`
3. 用报告内容替换模板占位符（`{{...}}`）
4. 浏览器本地打开自测：翻页、字号、断行、暗色背景可读性
5. `git add/commit/push`

## 五、展示规范

- 一页不超过 5 条 bullet
- 每条不超过 2 行
- 数字结论优先（涨跌幅、区间、阈值）
- 一句话结论必须出现在封面和结尾
- 如果是决策会，用“触发条件”替代模糊建议

## 六、风格变体（以后可扩展）

- `template-dark`：深色商务（当前）
- `template-light`：浅色打印友好
- `template-investor`：强调 KPI 卡片
- `template-minimal`：极简文字
