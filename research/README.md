# research

一个用于**项目调研**的工作目录，兼容两类场景：

- 软件开发项目（技术可行性、架构、成本、风险）
- 商业项目（市场、模式、财务、执行路径）

目标：在同一套流程下，既保持通用方法论，又保留不同项目类型的独特分析维度。

## 目录结构

- `templates/`：可复用模板
- `topics/`：按调研主题组织内容（每个主题一个子目录）
- `sources/`：原始资料与引用清单
- `notes/`：过程记录（建议按日期命名）
- `reports/`：对外或对内输出报告
- `templates/slide/`：HTML 幻灯片模板与固化流程

## 推荐流程

1. 在 `topics/` 新建主题目录
2. 复制 `templates/topic/` 下模板到主题目录
3. 先写 `01-overview.md`，明确目标与成功标准
4. 根据项目类型（技术/商业）填写对应章节
5. 汇总到 `reports/` 形成决策建议
6. 按 `templates/slide/README.md` 将报告同步产出同级 `.slide.html` 用于演示

## 主题目录建议

建议使用：

`topics/YYYY-MM-<主题关键词>/`

该目录内建议包含：

- `01-overview.md`
- `02-analysis.md`
- `03-decision.md`
- `04-next-actions.md`

## 通用原则

- 结论必须可追溯到 `sources/`
- 假设与事实要明确区分
- 每个结论附带风险与替代方案
- 决策文档要给出“为什么现在做/不做”
