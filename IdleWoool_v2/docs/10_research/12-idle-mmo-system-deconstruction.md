## 2. 系统拆解矩阵

## 2.1 战斗系统（Battling）

### 输入
- 角色战斗属性：`Attack Power / Protection / Agility / Accuracy / Damage / Crit`
- 敌人属性与等级
- 食物（提升有效生命）
- 姿态（Balanced / Offensive / Defensive / Agile / Dexterous）
- 敌人缩放设置（80+、100+会员阶段）

### 规则
- 一次只选 1 类敌人开战，持续打完整个敌人堆栈，直到：
  1) 角色被击败；
  2) 敌人堆栈清空；
  3) 达到最大挂机时长。
- 食物转化为有效生命：`EHP = HP + Σ(食物HP)`；提前结束会返还未消耗食物。
- 姿态决定属性经验分配方向；战斗等级经验与姿态经验并行获得。
- 每敌人最短击杀时间为 `8s`（硬下限）。
- 每场战斗最少承受 `1` 点伤害（硬下限）。
- 敌人缩放可提升经验与战利品收益；战利品增益通过 `Magic Find` 会话加成体现（最高示例到 +40%）。

### 输出
- 战斗经验、属性经验、掉落、金币（按敌人与机制）
- 食物消耗结果
- 战斗会话统计（命中、受伤、击杀剩余等）

### 边界
- 姿态战中不可切换，需中断后重开。
- 缩放存在会员与等级门槛，且为测试特性（文档标注 experimental）。
- `Magic Find` 提升的是“掉落表内物品概率权重”，不直接提高“是否掉落”的基础触发机会。

---

## 2.2 成长系统（Stats / Level / Effects）

### 输入
- 主属性等级：Strength / Defence / Speed / Dexterity
- 衍生属性：Attack Power / Protection / Agility / Accuracy / Damage / Crit
- 各类效果源：职业、会员、药水、活动、装备等

### 规则
- 主属性对衍生属性有线性映射（示例：每 1 点 Strength 增加 2.4 Attack Power）。
- Effect 采用可叠加合并口径：正负效果求和得到净加成。
- Efficiency 计算公式（硬规则）：
  - `Final Value = Initial Value / ((Efficiency% + 100) / 100)`
- Idle 行为在“开始时预计算收益”，途中新增效果不追溯到当前会话。

### 输出
- 角色战力变化（命中、伤害、承伤、行动效率）
- 经验增长与会话收益变化

### 边界
- 预计算机制导致“中途加 buff 不生效”是系统级限制。
- 部分历史系统已移除（如 Exhaustion），不可作为现行平衡基准。

---

## 2.3 装备/职业/技能系统（Classes + Combat Talents）

### 输入
- 角色职业选择（战斗型、生活型、挑战型）
- 角色 Combat Level
- 变职道具 Metamorphite 与变职冷却

### 规则
- 职业提供“永久效果 + 战斗天赋（按 Combat Level 阶段解锁）”。
- 典型战斗职业在 `10 / 35 / 70` 级解锁三段天赋。
- 变职需消耗道具，且同角色 `14 天` 冷却。
- `Forsaken / Cursed / Banished` 为锁定职业，无法变入/变出。

### 输出
- 属性成长倾向差异、战斗表现差异、玩法路径差异

### 边界
- 锁定职业无法通过常规系统回退。
- 部分职业无战斗天赋，定位偏生活/挑战体验。
- 市场可访问性受职业影响（Banished/Cursed 不可交易）。

---

## 2.4 交易系统（Market Listings / Orders）

### 输入
- 玩家上架条目（Listings）
- 玩家采购单（Purchase Orders）
- 会员状态、角色总等级（用于交易权限）

### 规则
- 市场由上架与订单双轨组成，采用自动匹配逻辑。
- 上架 `90 天` 未售自动下架。
- 下单收取基础费：免费用户 `15%`、会员 `12%`；订单时长附加费按档位递增。
- 基础下单费在订单过期时不退。
- 市场并发额度：免费 `10`、会员 `25`（上架+订单总量受限）。
- 小号需达到总等级 `150` 才可接入市场。

### 输出
- 交易成交、资金流转、物资再分配

### 边界
- 版本差异限制：玩家不可手动直接 fulfill 订单，需通过上架触发自动匹配。
- 市场权限由账号状态+角色条件共同约束。

---

## 2.5 地图/探索系统（Hunting + 区域门槛）

### 输入
- Hunting Mastery 等级
- Movement Speed
- 角色 Combat Level 与区域敌人等级
- 挂机时长上限

### 规则
- 先“狩猎发现敌人”再进入战斗，发现速率由 Hunting 与移动速度共同决定（乘性增益）。
- 文档提供发现速率框架：
  - `EnemiesPerSecond = BaseSpawnRate × (1 + NormLevel + NormMoveSpeed)`
  - 基础值示例：`BaseSpawnRate=0.03`。
- 角色只能稳定狩猎不高于自身 Combat Level 的敌人。
- 敌人会随时间流失（Enemy Decay）：高 Hunting 等级可降低流失比例。
- 可提前取消并按进度结算，后续可续接进度。

### 输出
- 可战斗敌人池规模
- 狩猎经验与后续战斗供给

### 边界
- 区域越级会导致“可发现敌人不足或为零”。
- 敌人存在时间窗口，未及时处理将自然流失。

---

## 2.6 任务/活动系统（Quests）

### 输入
- 所在地点（必须到对应区域）
- 前置任务完成数
- 任务目标物资（可来自战斗/生产等）

### 规则
- 任务按地点与链路推进，区域解锁依赖累计完成任务数（如 0/3/6/10/.../40）。
- 当前任务主要为“收集并交付物品”。
- 多数后续任务有冷却；冷却起点与前序任务完成时间相关。
- 完成任务时扣除需求物品并发放奖励。

### 输出
- 金币、物品、经验
- 新地点/新任务链解锁

### 边界
- 地图门槛与任务链门槛叠加形成“线性主进度阀门”。
- 背包容量影响交付稳定性（满包可能影响奖励投递）。

---

## 2.7 地牢系统（Dungeons）

### 输入
- 地牢等级、难度值、门票成本、单次时长
- 角色 Combat Stats（AP/Protection/Agility/Accuracy 汇总）
- 角色血量与可连续运行能力

### 规则
- 入场门槛：角色 Combat Stats 至少达到地牢难度 `70%`。
- 通关后按“角色战力 vs 地牢难度”结算生命损耗。
- 当战力高于难度 `30%+` 可获得 Magic Find 奖励。
- 可链式运行直至挂机上限，但必须满足连续血量可承受。
- 通关可累积“永久 Magic Find”进度（示例 +1%/完成一个地牢）。
- 高等级地牢产出 shards，可在地牢商店兑换资源。

### 输出
- 地牢经验、配方、碎片、长期 Magic Find 成长

### 边界
- 典型高收益玩法受“时间+金币”双成本约束。
- 链地牢受血量约束，非纯时长约束。

---

## 2.8 资源产消系统（Food / Gold / Token / Magic Find）

### 输入
- 产出来源：战斗、地牢、世界 Boss、任务、日常奖励
- 消耗来源：地牢门票、订单费、道具投入、时间投入
- 调节变量：Magic Find、效率、会员、职业特性

### 规则
- Food 作为战斗续航资源，直接转化 EHP，未使用可返还（有优先消耗弱食物倾向）。
- Gold 在地牢、市场费用中持续回收，形成货币 sink。
- Token 与 Daily Reward 用于回流与 streak 恢复（示例：断签 48h 内 300 token 可恢复 streak）。
- Daily Reward 每日 12:00 UTC 刷新，奖励周期 1~60 天，连签提供分段 Magic Find（会员可扩展到更高档）。
- Magic Find 与多系统耦合（战斗/地牢/Boss/日常），但需经过“总掉率超额裁剪”机制防止爆表。

### 输出
- 玩家收益曲线（短期回流 + 中长期稀缺追求）
- 经济活性与资源循环稳定性

### 边界
- 高 Magic Find 不应突破掉落系统上限，系统会裁剪常见掉落权重。
- 多来源叠加必须经统一结算口径，否则易出现通胀或稀有度坍塌。

## 3. 跨系统耦合图（v2 可复用）

1. **探索 → 战斗**：Hunting 决定敌人供给上限，战斗决定单位时间产出。
2. **成长 → 战斗/地牢**：属性与效果决定通关效率和损耗曲线。
3. **战斗/地牢 → 经济**：掉落与材料进入市场形成供给。
4. **日常/活动 → Magic Find**：回流行为影响高价值掉落概率。
5. **职业/权限 → 经济参与度**：市场访问权限改变账号资源流动能力。

## 4. 给 IdleWoool_v2 的可执行输入（面向后续 TP-D/TP-S）

1. 在 `TP-D2` 核心循环中显式拆分“发现敌人层（Hunting）”与“结算战斗层（Battling）”。
2. 在 `TP-D3` 成长框架中采用“主属性→战斗派生属性→活动效率”的三段映射。
3. 在 `TP-S4` 交易经济设计中保留“基础费 + 时长费 + 权限门槛 + 自动撮合”四件套。
4. 在 `TP-S1/TP-S3` 中加入硬边界：最短击杀时长、最小承伤、掉率裁剪、防过载机制。
