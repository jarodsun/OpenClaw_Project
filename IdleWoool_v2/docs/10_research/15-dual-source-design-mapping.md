## 1. 融合方法与判定口径

1. **产品可执行优先**：优先保留可直接支撑 Web 挂机主循环的机制。
2. **传奇语义优先植入**：在不破坏挂机效率模型前提下，优先注入传奇世界观、命名、职业语义与社会竞争感。
3. **系统稳定优先**：对高复杂、高同步成本、强 PVP 实时依赖机制，在 v2 只保留设计接口，不强行继承。
4. **三类映射标准**：
   - 可直接继承：无需结构改造即可进入 v2 设计文档。
   - 需变体改造：保留内核但需改写为 Idle/Web 形态。
   - 不采用：与 v2 目标冲突或风险过高，暂不纳入。

---

## 2. 可直接继承清单（Adopt As-Is）

### 2.1 主循环骨架（来自 Idle MMO）
- 发现敌人（探索）→ 战斗结算 → 资源产出 → 成长/交易 → 回流再投入。
- 继承原因：与 v2 目标（Web 挂机）完全同构，可直接进入 `TP-D2`。

### 2.2 成长三段映射（来自 Idle MMO）
- 主属性 → 战斗派生属性 → 活动效率。
- 继承原因：规则可参数化，便于 `TP-D3` 与后续 `TP-S1/S2` 对齐。

### 2.3 资源产消闭环（双源共识）
- 战斗/活动产出 + 消耗端回收（门票、手续费、强化消耗）。
- 继承原因：既符合 Idle MMO 经济闭环，也契合传奇“打宝-消耗-再打宝”的长期动机。

### 2.4 三职业非对称定位（来自传奇）
- 战士（承压/爆发）、法师（范围/效率）、道士（召唤/续航）。
- 继承原因：职业语义清晰、可持续扩展，直接作为 `TP-S2` 基础。

### 2.5 术语与命名风格（来自传奇语料库）
- 打宝、爆装、行会、攻城、卷轴、道符等核心词典。
- 继承原因：提升目标用户识别度，可直接用于 `TP-D1` 文案规范。

## 3. 需变体改造清单（Adapted Variants）

### 3.1 行会/公会竞争
- 原型来源：Idle MMO Guild Conquest + 传奇攻城文化。
- 变体方向：
  - v2 不做强实时攻防；改为“异步赛季占领 + 时段结算”。
  - 保留行会协作、据点争夺、赛季排名。
- 进入任务包：`TP-S4`、`TP-B2`。

### 3.2 地图与区域推进
- 原型来源：传奇分层地图 + Idle MMO 探索供给。
- 变体方向：
  - 区域以“挂机推荐战力区间 + 风险收益梯度”表达。
  - 减少纯手动跑图，强化“区域选择策略”。
- 进入任务包：`TP-S5`。

### 3.3 掉落与极品体验
- 原型来源：传奇打宝峰值体验 + Idle MMO 掉率系统。
- 变体方向：
  - 保留稀有掉落与词条峰值体验。
  - 引入保底、衰减与掉率裁剪，避免经济失衡。
- 进入任务包：`TP-S3`、`TP-B1`。

### 3.4 技能构筑深度
- 原型来源：传奇职业技能语义与分支选择。
- 变体方向：
  - v2 采用“少量主动配置 + 自动执行”为主。
  - 保留技能路线分歧，不做高频手操连招。
- 进入任务包：`TP-S2`。

### 3.5 市场交易机制
- 原型来源：Idle MMO 自动撮合市场 + 传奇摆摊文化。
- 变体方向：
  - 主体采用自动撮合（可控、可审计）。
  - 摆摊语义保留为“展示层/频道层”，不引入实时手动议价核心链路。
- 进入任务包：`TP-S4`。

---

## 4. 不采用清单（Excluded in v2）

### 4.1 强实时大规模同屏攻防
- 不采用原因：实现复杂度高、服务器与同步成本高，偏 v5 工程议题。

### 4.2 高自由度无保护玩家互害机制
- 不采用原因：新手体验风险高，易破坏挂机节奏与留存。

### 4.3 无边界开放交易（零税/零门槛）
- 不采用原因：极易被脚本与套利破坏，无法维持健康经济。

### 4.4 完全手动战斗操作链
- 不采用原因：与 v2“挂机为主、策略为辅”定位冲突。

---

## 5. v2 设计输入约束（供 TP-D* / TP-S* 直接使用）

1. **循环约束**：任何系统必须回到“挂机收益可解释”的主循环，不允许孤立玩法。
2. **语义约束**：命名与叙事优先使用传奇语料库，不得引入风格割裂词汇体系。
3. **数值约束**：所有高价值产出都必须绑定回收点（金币/材料/耐久/手续费）。
4. **职业约束**：三职业保持非对称，禁止仅数值倍率区分。
5. **交易约束**：交易系统必须包含门槛、费用、反操纵规则三件套。
6. **社交约束**：行会系统需影响收益或成长效率，不能仅聊天装饰。
7. **风险约束**：涉及强对抗内容，优先异步化、赛季化，不引入高实时硬依赖。
