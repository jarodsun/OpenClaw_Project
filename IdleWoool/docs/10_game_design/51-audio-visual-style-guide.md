# 51 美术与音频风格指南（Audio/Visual Style Guide）

- 更新时间：2026-03-07
- 适用阶段：MVP（文档优先，先定可执行基线）
- 关联文档：`50-ui-ux-guidelines.md`、`20-balance-framework.md`、`42-observability-and-logs.md`、`44-release-plan-and-milestones.md`

## 1. 目标与范围

### 1.1 目标

为 IdleWoool 建立统一的视觉与音频表达规则，保证以下结果：

1. 玩家在 5 分钟内形成“暗黑奇幻 + 挂机成长”的稳定感知；
2. 关键反馈（掉落、升级、暴击、失败、可领取）可在 300ms 内被识别；
3. 前端资源实现与性能预算可量化、可验收、可回滚。

### 1.2 In Scope（MVP）

1. UI 视觉语法（颜色、字体、间距、动效节奏）；
2. 战斗与养成核心反馈特效（命中、暴击、掉落、强化、离线领奖）；
3. BGM 与 SFX 的分层、混音、触发规则；
4. 资源命名、切图规格、压缩策略、性能预算。

### 1.3 Out of Scope（非 MVP）

1. 全剧情过场动画与配音；
2. 大规模角色 Live2D/骨骼动画方案；
3. 多语言语音包与区域化音频重混。

## 2. 视觉风格基线

### 2.1 关键词与情绪板

- 关键词：暗黑奇幻、粗粝金属、微暖火光、成长爽感、克制炫光。
- 情绪比例：
  - 基础氛围（沉稳）60%
  - 战斗刺激（高亮）25%
  - 收益奖励（庆祝）15%

### 2.2 色彩系统（Token）

1. 中性色（背景/容器）
   - `bg-900`：#0F1115
   - `bg-800`：#171A21
   - `surface-700`：#1F2430
2. 语义色
   - `primary`（主强调）：#D4A24C
   - `success`（正向）：#5FC77B
   - `warning`（提示）：#E7A93A
   - `danger`（失败）：#D95C5C
   - `info`（信息）：#5EA8E5
3. 稀有度色（与 `12-system-equipment-and-loot.md` 对齐）
   - 普通：#9AA4B2
   - 精良：#4FA3FF
   - 稀有：#8E6BFF
   - 史诗：#C94CFF
   - 传说：#FFB347

约束：

- 单屏高饱和颜色面积占比不超过 25%；
- 红/绿信息必须叠加图标或文案，避免仅靠颜色传达状态。

### 2.3 字体与排版

1. 字体族：系统无衬线优先（`Inter`/`PingFang SC` 回退）；
2. 字号梯度：12 / 14 / 16 / 20 / 24 / 32；
3. 行高：1.4~1.6；
4. 数值显示：战斗数字与收益数字使用等宽数字（tabular figures）。

### 2.4 图标与插画

1. 图标网格：24px 基线，描边 1.5px；
2. 设备图标风格统一：圆角 2px，避免混用拟物与极简；
3. 装备图标至少支持 5 档稀有度外框叠层；
4. 关键资源导出：`webp` 优先，必要透明素材使用 `png`。

## 3. 动效与战斗表现

### 3.1 动效节奏

1. 微交互（hover/press）：80~150ms；
2. 状态切换（面板开关）：180~240ms；
3. 反馈强化（暴击/掉落）：220~320ms；
4. 页面级转场：不超过 300ms。

### 3.2 战斗反馈分级

1. 低强度反馈：普通命中（轻闪 + 小幅位移）；
2. 中强度反馈：技能命中（方向性拖尾 + 命中音）；
3. 高强度反馈：暴击/击杀（屏幕局部震动 + 高亮数字 + 粒子爆发）。

约束：

- 同屏高强度反馈并发不超过 2 个；
- “减少动态效果”开启后，震动与大幅闪烁改为静态高亮。

### 3.3 离线奖励表现

1. 离线结算弹层优先展示：离线时长、核心资源、稀有掉落；
2. 领取动效时长不超过 500ms，支持“跳过动画立即入账”；
3. 与 `13-system-offline-rewards.md` 的发放幂等规则一致，不因动效失败阻塞结算。

## 4. 音频风格与实现

### 4.1 音频分层

1. BGM：场景持续氛围（主城/战斗/领奖）；
2. SFX：即时反馈（点击、命中、暴击、掉落、强化、升级）；
3. UI 音：界面操作短音（打开、确认、返回、错误）。

### 4.2 音色与混音目标

1. BGM 风格：暗色合成器 + 低频打击，避免旋律过饱和；
2. SFX 优先中高频清晰度，避免与 BGM 低频冲突；
3. 默认混音建议：
   - Master：0 dB
   - BGM：-14 dB
   - SFX：-10 dB
   - UI：-12 dB

### 4.3 触发与防噪约束

1. 连击类 SFX 需做 80ms 防抖，避免短时噪声堆叠；
2. 同类音效并发上限 4，超过时丢弃最旧实例；
3. 切后台自动静音，回前台平滑恢复（150ms）；
4. 支持独立音量滑杆：总音量/BGM/SFX。

### 4.4 音频可访问性

1. 关键音频反馈（暴击、可领取、失败）必须有对应视觉提示文案或图标；
2. 静音状态下，不影响核心流程判断（奖励可领、操作成功/失败均可从 UI 直接识别）；
3. 禁止仅通过音高变化传递关键状态。

## 5. 资源规范与性能预算

### 5.1 命名规范

统一命名：`<module>_<type>_<name>_<variant>_<v>`

示例：

- `battle_sfx_crit_hit_a_v1.wav`
- `ui_icon_loot_epic_v2.webp`

### 5.2 格式与预算

1. 图片：
   - UI 图标：`webp`，单资源建议 < 80KB；
   - 大图背景：`webp`，单资源建议 < 400KB；
2. 音频：
   - SFX：`wav`（短音）或 `ogg`（压缩分发）；
   - BGM：`ogg`，单曲建议 < 2.5MB（移动端优先）；
3. 首屏 A/V 预算：
   - 首屏视觉资源总量 ≤ 1.5MB；
   - 首屏可感知音频资源 ≤ 600KB；
   - 首屏加载 + 可交互时间目标与 `50-ui-ux-guidelines.md` 保持一致。

### 5.3 降级策略

1. 低性能设备：关闭粒子层 2、减少阴影模糊半径；
2. 弱网首启：优先加载核心 UI/SFX，BGM 延迟加载；
3. 音频上下文初始化失败：不阻塞游戏流程，记录告警并允许玩家重试初始化。

## 6. 产物清单与验收（DoD）

### 6.1 必交付物

1. 视觉规范页（颜色、字体、间距、图标样式）；
2. 关键页面高保真稿（主界面、战斗、背包、离线领奖）；
3. 动效规范表（触发条件、时长、并发限制）；
4. 音频清单（BGM/SFX/UI 音）与触发映射；
5. 资源清单（命名、格式、大小、版本号）。

### 6.2 验收标准

1. 设计与实现视觉偏差（关键页面）≤ 10%；
2. 关键反馈可识别率（可用性走查）≥ 95%；
3. 首屏性能预算满足 5.2 约束；
4. “减少动态效果”与静音/分音量能力可用；
5. 抽检 20 个资源文件，命名与格式合规率 100%。
