# 50 UI/UX Guidelines（MVP）

更新时间：2026-03-07

## 1. 目标与范围

### 1.1 目标
- 为 IdleWoool Web 版提供可直接落地的 UI/UX 基线，确保“挂机主循环”在 PC 与移动端浏览器都能顺畅完成。
- 降低首日理解成本，让用户在 3 分钟内完成“进入战斗 -> 获得奖励 -> 装备强化 -> 离线结算领取”的核心闭环。
- 统一前端交互、视觉层级、反馈节奏与异常提示，减少联调期返工。

### 1.2 适用范围（MVP）
- In Scope：主界面 HUD、战斗信息区、背包与装备页、技能页、离线奖励弹窗、设置页、登录与会话状态提示。
- Out of Scope：剧情演出系统、复杂新手引导动效、社区聊天 UI、赛季活动专题页。

## 2. 信息架构与导航

### 2.1 一级导航
- 底部固定 5 Tab：`战斗`、`角色`、`背包`、`技能`、`设置`。
- 当前 Tab 必须高亮并显示图标+文字；不可只用颜色区分。
- 任一页面返回主战斗页路径不超过 2 次点击。

### 2.2 页面优先级
- P0：战斗页（默认进入页）。
- P1：离线奖励弹窗、背包装备更换。
- P2：技能页、设置页。

### 2.3 关键任务路径（首会话）
1. 进入战斗页，看到当前关卡、战力、收益速率。
2. 点击领取离线收益（若有），确认到账反馈。
3. 打开背包，完成一次装备穿戴或强化。
4. 返回战斗页观察战力变化并继续挂机。

## 3. 关键页面规范

### 3.1 战斗页
- 顶栏：账号、资源（金币/钻石）、网络状态、设置入口。
- 中区：敌人信息、当前 DPS、战斗日志摘要（最近 3 条）。
- 底区：关卡进度条、挂机收益速率、主操作按钮（挑战/领取/切换自动）。
- 数据刷新节奏：核心战斗数值 1s 刷新；资源区 2s 刷新；异常状态即时刷新。

### 3.2 背包与装备页
- 默认排序：稀有度降序 + 战力收益降序。
- 装备对比必须展示“当前装备 vs 候选装备”的三列差值（攻击、防御、战力）。
- 危险操作（分解/重置）必须二次确认，并展示预计返还资源。

### 3.3 技能页
- 技能卡片必须显示：类型、等级、下一级收益、冷却/触发条件。
- 升级按钮禁用时必须给出具体原因（资源不足、前置未达成、等级上限）。
- 被动技能默认展开“当前生效口径”（快照/实时），与 `11-system-professions-and-skills.md` 一致。

### 3.4 离线奖励弹窗
- 打开时展示离线时长、基础收益、效率系数、最终到账。
- 使用默认口径：8h 上限、70% 效率（未拍板前沿用 `Q-001` 默认方案）。
- 必须提供“领取并返回战斗页”主按钮，次按钮为“稍后处理”。

## 4. 交互与反馈规范

### 4.1 状态反馈
- 点击反馈：按钮按下态 <= 100ms 内可见。
- 成功反馈：领取/升级后 300~800ms 内给出数值飘字或 toast。
- 失败反馈：错误提示必须包含“原因 + 建议动作”，禁止仅显示“失败”。

### 4.2 加载与空态
- 页面加载超过 300ms 显示骨架屏，超过 2s 显示可重试提示。
- 空背包、无离线收益等空态需含“下一步行动”按钮。
- 弱网重试默认 2 次指数退避（300ms, 900ms），仍失败则展示离线降级文案。

### 4.3 异常与降级
- 配置拉取失败：使用本地缓存配置并提示“使用缓存版本”。
- 战斗流异常：不阻塞主页面渲染，展示“战斗数据延迟更新”状态条。
- 埋点上报失败：静默重试，不打断玩家操作。

## 5. 可用性与可访问性

### 5.1 文案与术语
- 与系统文档术语一致：关卡、战力、离线收益、幂等领取。
- 中文文案优先短句，单条 toast 不超过 22 个汉字。
- 数值文案统一格式：百分比保留 1 位小数，千位分隔。

### 5.2 对比与颜色
- 正向收益：绿色；负向变化：红色；中性信息：灰蓝色。
- 不可仅依赖颜色表达状态，需配合图标或文本标签。
- 关键文本与背景对比度建议不低于 WCAG AA（4.5:1）。

### 5.3 触控与动效
- 主要可点击区域最小尺寸：`44px × 44px`。
- 可滚动区域与主操作按钮需留出安全间距，避免误触。
- 支持“减少动态效果”系统偏好：关闭非必要位移与闪烁动画，仅保留必要状态反馈。

## 6. 前端实现约束（Next.js）

### 6.1 组件分层
- 页面级：`app/(game)/*/page.tsx` 仅做编排，不放业务计算。
- 领域组件：`components/game/*` 承载战斗、背包、技能模块。
- 共享组件：`components/ui/*` 统一按钮、弹窗、toast、骨架屏。

### 6.2 状态管理
- 服务端状态：使用查询层统一管理（缓存、重试、失效）。
- 客户端临时状态：仅放 UI 控制态（弹窗开关、筛选条件）。
- 禁止在多个组件重复计算同一战力派生值，统一使用 selector。

### 6.3 埋点最小集
- 必埋事件：`offline_claim_click`、`equip_change`、`skill_upgrade`、`battle_page_enter`。
- 每个事件必须包含：`user_id`、`session_id`、`zone_id`、`latency_ms`。

## 7. 验收标准（DoD）
- 主循环关键路径（战斗->领取->装备->返回）可在 3 分钟内完成。
- P0 页面首屏可交互时间（TTI）<= 2.5s（中端移动设备，4G）。
- 关键操作反馈成功率 >= 99%（排除网络硬中断）。
- UI 文档规则可映射到前端组件清单与测试用例（见 `41-testing-strategy.md`）。
