# 核心循环设计（挂机/成长/反馈）

> 状态：已完成（v1.0）
> 更新时间：2026-03-07

## 1. 文档目标

将 PRD 中“打怪-掉落-变强”的产品目标落地为可执行玩法循环，明确状态机、结算口径、关键参数与验收条件，供系统设计与研发实现直接使用。

## 2. 循环定义与边界

### 2.1 主循环（在线）

1. 进入区域：玩家选择或自动进入当前可挂机区域。
2. 自动战斗：按战斗 tick 执行普攻/技能与怪物行为。
3. 产出结算：按击杀或时间结算经验、金币、掉落。
4. 成长反馈：触发升级、战力变化、装备替换、技能提升提示。
5. 推进解锁：达到条件后解锁更高区域，进入下一轮收益更高循环。

### 2.2 离线循环

1. 记录离线起止时间与离线前区域状态。
2. 按离线收益规则进行模拟结算（受离线时长上限约束）。
3. 登录后一次性展示收益汇总与关键事件（升级/关键掉落）。
4. 应用收益后回到在线主循环。

### 2.3 边界约束

- 核心循环必须在低操作下持续推进，不依赖高频手动点击。
- 在线与离线收益口径一致，仅效率和上限存在策略差异。
- 任意时刻都应有短期目标（升级/换装）与中期目标（区域推进）。

## 3. 状态机设计（MVP）

```text
Idle(待机)
  -> EnterZone(进入区域)
  -> AutoCombat(自动战斗)
  -> Settle(结算)
  -> Growth(成长反馈)
  -> CheckUnlock(解锁检查)
  -> AutoCombat / EnterZone

Logout
  -> OfflineSim(离线模拟)
  -> OfflineSettle(离线结算)
  -> EnterZone
```

状态说明：

- `AutoCombat`：仅处理战斗行为，不直接修改背包展示。
- `Settle`：统一处理经验、金币、掉落写入，保证事务一致性。
- `Growth`：集中处理升级、自动穿戴、战力重算与提示。
- `CheckUnlock`：负责区域推进判定，避免散落在多个系统中。

## 4. 核心规则口径

### 4.1 战斗与产出

- 战斗最小结算粒度：`tick = 1s`（MVP 建议）。
- 怪物刷新：按区域固定刷新池循环抽取。
- 经验产出：`base_exp(zone, monster) * exp_multiplier`。
- 金币产出：`base_gold(zone, monster) * gold_multiplier`。
- 掉落判定：先判定掉落池，再判定品质与词条。

### 4.2 成长反馈

- 升级后即时应用基础属性增长。
- 装备自动穿戴策略：仅当综合战力提升为正时替换。
- 技能自动释放遵循冷却与资源约束。
- 战力展示口径统一为“角色属性 + 装备 + 技能加成”的实时快照。

### 4.3 区域推进

- 解锁条件至少包含：角色等级阈值 + 前置区域完成条件。
- 新区域收益期望应高于上一层级，避免推进负收益。
- 允许手动回退低级区域刷取特定资源。

### 4.4 离线收益

- 离线时长上限：默认 `8h`（待平衡文档确认最终值）。
- 离线效率系数：默认 `70%`（相对在线理论收益）。
- 离线结算不触发实时战斗事件，仅结算结果与关键里程碑。
- 离线奖励领取必须幂等，防止重复发奖。

## 5. 数值与配置入口

本循环依赖以下配置域，具体 schema 在 `21-content-config-schema.md` 定义：

- 区域配置：等级门槛、怪物池、基础收益倍率。
- 怪物配置：基础属性、经验/金币基值、掉落池映射。
- 掉落配置：装备类型、品质权重、词条池。
- 离线配置：时长上限、效率系数、补偿规则。
- 自动策略配置：自动穿戴阈值、技能释放策略。

## 6. 失败与异常处理

- 结算失败：回滚本次结算事务并记录错误日志。
- 配置缺失：回退至保底配置并标记告警。
- 战力计算异常：使用上次稳定快照，避免阻塞挂机。
- 离线结算超时：分段结算并在前端提示“部分到账”。

## 7. 验收标准（实现前评审）

满足以下条件即视为“核心循环文档可用于开发”：

1. 在线循环与离线循环流程、状态机、结算边界已明确。
2. 关键口径（tick、离线上限、离线效率）有默认值且可配置。
3. 与 PRD 的体验目标一致，不引入超出 MVP 的重交互设计。
4. 可直接映射到后续文档：
   - 战斗成长：`10-system-combat-and-progression.md`
   - 离线系统：`13-system-offline-rewards.md`
