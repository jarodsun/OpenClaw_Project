# Idle Web Game
  
This is an idle web game project.
  
## 游戏设计建议 (借鉴《传奇世界》)
 
**核心挂机循环：**
  
1.  **自动打怪升级：**
    *   玩家角色自动在地图上寻找并攻击怪物。
    *   击败怪物获得经验、金币和掉落物。
    *   经验值累积，角色自动升级，提升基础属性。
    *   可以设置“挂机区域”，玩家选择在哪个区域挂机，不同区域怪物强度和掉落不同。
2.  **离线收益：**
    *   玩家离线期间，角色继续自动打怪，积累经验、金币和物品。
    *   上线时，一次性结算离线收益，让玩家有“收获”感。
    *   可以设置离线收益上限，或通过VIP/特权提升上限。

**角色与职业：**
 
1.  **经典三职业：**
    *   **战士：** 高生命、高防御、近战物理输出。挂机特点：稳定、生存能力强。
    *   **法师：** 低生命、高魔法攻击、群体伤害。挂机特点：清怪效率高。
    *   **道士：** 召唤宠物、辅助治疗、毒术。挂机特点：均衡、有宠物协助。
    *   每个职业可以有独特的挂机技能或天赋树。
 
**装备系统：**
  
1.  **自动穿戴：**
    *   角色自动拾取掉落的装备，并自动穿戴属性更好的装备。
    *   可以设置“锁定”功能，防止好装备被自动替换。
2.  **装备强化/精炼：**
    *   消耗金币或特定材料，提升装备的基础属性。
    *   强化等级可以有失败率，增加刺激感（但作为挂机游戏，失败惩罚不宜过重）。
3.  **极品属性/词条：**
    *   装备除了基础属性，还可以随机生成额外的“极品”词条（如攻击速度、暴击率、吸血等），增加刷装备的乐趣。
4.  **套装效果：**
    *   收集特定套装部件，激活额外的套装属性加成。
  
**技能系统：**
  
1.  **自动施放：**
    *   职业技能在战斗中自动施放。
    *   玩家可以设置技能施放优先级或冷却时间。
2.  **技能升级：**
    *   消耗经验、金币或技能书来提升技能等级，增强效果。
    *   可以有“被动技能”和“主动技能”，被动技能常驻，主动技能自动施放。
  
**特色玩法借鉴：**
  
1.  **打宝地图/BOSS：**
    *   设置一些特殊的“打宝地图”或“世界BOSS”，怪物更强，但掉落稀有装备和材料的几率更高。
    *   可以设计成定时刷新，或需要消耗特定道具才能进入。
    *   BOSS战可以有简单的动画表现，让玩家感受到挑战性。
1.  **行会/公会：**
    *   玩家可以创建或加入行会。
    *   行会可以提供成员额外的挂机加成（如经验、金币、掉落率）。
    *   行会任务：成员共同完成一些简单的任务，为行会积累资金或贡献度。
    *   **沙巴克攻城（简化版）：** 可以设计成一个周期性的行会PVP活动。例如，行会成员在活动期间贡献“战力值”，最终战力值最高的行会获得“沙巴克”称号和特殊奖励/BUFF。战斗过程可以简化为数值对抗。
1.  **PK（简化版）：**
    *   可以有“竞技场”或“排行榜”功能，玩家可以挑战其他玩家的离线角色。
    *   战斗过程自动进行，根据双方属性和装备计算胜负。
    *   胜利者获得排名提升和奖励。
1.  **挖矿/采集：**
    *   设置专门的资源点，玩家角色可以自动前往挖矿或采集草药。
    *   获得的资源用于装备强化、药品制作或出售。
  
**界面与体验：**
  
1.  **简洁明了的UI：** 挂机游戏的核心是展示收益和进度，UI要清晰地显示经验条、金币、背包、当前挂机区域等信息。
2.  **进度条与数字跳动：** 实时显示经验、金币、伤害数字的增长，给玩家直观的反馈。
3.  **成就系统：** 达成特定目标（如击杀10000只怪物、强化装备到+10）获得奖励。
4.  **邮件系统：** 用于发送离线收益报告、活动奖励等。
  
**技术栈建议：**
  
*   **前端：** HTML5, CSS3, JavaScript (可以考虑使用Vue/React/Svelte等框架简化开发)。
*   **后端（如果需要）：** Node.js (Express), Python (Flask/Django), Go等，用于处理用户数据存储、排行榜、行会等功能。对于纯客户端挂机游戏，可以不依赖后端。
*   **数据存储：** LocalStorage (客户端存储), IndexedDB (客户端存储), 或后端数据库 (如MongoDB, PostgreSQL)。
  
**开发流程建议：**
  
1.  **核心循环：** 先实现最基础的自动打怪、经验金币获取、升级。
2.  **装备系统：** 接着实现装备掉落、自动穿戴、强化。
3.  **职业技能：** 实现各职业的自动技能施放和升级。
4.  **特色玩法：** 逐步加入打宝、行会、PK等。
 
记住，挂机游戏的关键在于“放置”和“成长感”。让玩家在不投入大量时间的情况下，也能感受到角色在不断变强。
