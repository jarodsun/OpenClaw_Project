# IdleWoool

基于 Web 技术实现的放置挂机 RPG 项目（灵感来源于经典传奇系成长体验）。

## 项目目标

- 打造“低操作负担、高成长反馈”的挂机体验
- 优先实现可持续增长的核心循环：战斗 -> 掉落 -> 强化 -> 变强
- 支持离线收益结算与自动化战斗流程

## 核心玩法（MVP）

1. 自动打怪升级
- 角色自动寻敌、攻击、结算收益
- 怪物掉落经验、金币、装备
- 支持挂机区域切换（不同强度/掉落）

2. 离线收益
- 离线期间累计战斗收益
- 登录后一键结算离线奖励
- 设置离线上限与扩展机制

3. 职业与成长
- 三职业（战士/法师/道士）基础框架
- 职业差异化属性与技能优先级

4. 装备与强化
- 自动拾取与自动穿戴
- 装备强化/精炼
- 词条与套装效果（后续迭代）

## 技术方向

- 前端：`HTML5 + CSS3 + JavaScript`（可升级到 Vue/React）
- 数据：优先本地存档（`LocalStorage/IndexedDB`），后续按需接后端
- 后端（可选）：Node.js / Python / Go，用于排行榜、公会、活动等在线能力

## 目录结构

- `docs/`：需求与设计文档（包含原始策划参考）
- `src/`：游戏前端主代码
- `assets/`：图片、音效、UI 资源
- `data/`：配置与平衡参数
- `scripts/`：工具脚本（构建、导出、检查）

## 开发里程碑建议

- Phase 1：基础挂机循环（战斗、经验、金币、升级）
- Phase 2：装备系统（掉落、穿戴、强化）
- Phase 3：职业技能（自动施放、升级）
- Phase 4：扩展玩法（BOSS、公会、竞技场）
