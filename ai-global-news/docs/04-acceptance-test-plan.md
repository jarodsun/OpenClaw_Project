# 04 Acceptance Test Plan（MVP 验收测试计划）

## 1. 测试范围

- 采集、清洗、去重、摘要、检索、前端主链路

## 2. 环境前置

- SQLite3 初始化完成
- 至少接入 20 个来源
- 测试数据规模 >= 10,000 条

## 3. 用例清单

### A. 功能验收

1. 采集任务执行
- 输入：触发一次全量抓取
- 期望：任务成功并入库，失败任务可重试

2. 去重逻辑
- 输入：同一资讯多来源重复样本
- 期望：重复内容正确归并/标注

3. 标签分类
- 输入：模型发布、融资、开源三类样本
- 期望：标签命中率达到预期（初期人工抽检 >=80%）

4. 摘要生成
- 输入：随机 200 条英文资讯
- 期望：中文摘要成功率 >= 90%

5. API 查询
- 输入：关键词 + 标签 + 时间过滤
- 期望：接口返回正确、分页正常

6. 前端主链路
- 输入：访问列表 -> 详情 -> 标签筛选
- 期望：链路无阻塞、页面可用

### B. 性能验收

7. 检索性能
- 条件：1 万条数据，10 并发
- 期望：P95 <= 800ms

### C. 稳定性验收

8. 连续运行稳定性
- 条件：连续运行 72 小时
- 期望：关键任务成功率 >= 95%，无致命崩溃

## 4. 验收门禁（Go/No-Go）

- 必须全部满足：
  - 抓取成功率 >= 95%
  - 摘要成功率 >= 90%
  - 重复占比 <= 15%
  - 检索 P95 <= 800ms
  - 72 小时稳定运行通过
