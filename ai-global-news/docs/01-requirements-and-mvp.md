# AI Global News - 需求与 MVP 验收标准

## 1. 项目目标

构建一个可持续运行的“全球 AI 资讯聚合系统”，实现从多源采集到结构化输出的全链路能力，支持检索、筛选、订阅与推送。

## 2. 核心用户与场景

- 个人研发者：快速获取高价值 AI 动态，减少信息噪音。
- 内容运营者：基于聚合结果生成日报/周报。
- 技术团队：跟踪模型、开源、投融资、政策变化。

## 3. 功能范围

### 3.1 MVP 必须具备

- 多源采集（至少 20 个来源，覆盖新闻/论文/开源）
- 统一清洗（标题、正文、发布时间、来源）
- 去重（同链接 + 高相似标题）
- 自动标签（模型发布、开源、融资、政策、应用）
- 中文摘要（短摘要）
- API 查询（列表、详情、关键词搜索、按标签过滤）
- Web 资讯流页面（列表 + 详情）

### 3.2 第二阶段（非 MVP）

- 个性化订阅（关键词、标签、来源组合）
- 邮件/Telegram/Discord 推送
- 向量检索与“相似资讯推荐”
- 趋势分析看板（周度主题热度）

## 4. 非功能要求

- 稳定性：抓取任务成功率 >= 95%
- 时效性：重点来源 30~60 分钟内可见
- 可观测性：抓取、清洗、摘要、入库各环节有日志和错误计数
- 可维护性：采集器插件化，新增来源不改主流程

## 5. 数据口径

每条资讯最少包含：

- `id`
- `source_name`
- `source_url`
- `title`
- `url`
- `published_at`（UTC）
- `language`
- `content_text`
- `summary_zh`
- `tags[]`
- `dedup_key`
- `created_at`

## 6. MVP 验收标准（硬指标）

- 来源数量：>= 20 个有效来源
- 去重效果：重复内容占比 <= 15%
- 摘要成功率：>= 90%
- 抓取成功率：>= 95%
- 检索性能：关键词查询 P95 <= 800ms（1 万条数据量级）
- 页面可用：资讯列表、详情、标签筛选可正常使用

## 7. 里程碑（建议）

- M1（第 1 周）：采集 + 入库 + 基础 API
- M2（第 2 周）：去重 + 标签 + 摘要
- M3（第 3 周）：前端页面 + 监控 + 验收

## 8. 风险与应对

- 反爬限制：优先 RSS/API，网页采集设置限速与重试
- 低质量来源：引入来源信誉评分，支持黑白名单
- 摘要成本：支持批处理、缓存、失败降级策略
- 字段不统一：统一 Canonical Schema + 映射层
